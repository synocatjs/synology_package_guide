# 桌面应用程序

您可以为套件提供应用程序配置，这样配置的应用程序将显示在桌面的菜单中。可以自定义图标、应用程序权限和目标 URL。

为了区分不同角色的用户，一个套件甚至可以提供多个应用程序，例如为管理员提供管理应用程序，为普通用户提供普通应用程序。

---

## DSM Web GUI

此外，任何应用程序都可以通过提供帮助配置，将其帮助文档集成到桌面中。

---

## 设置桌面应用程序的步骤

### 1. 创建配置目录

在 `package.tgz` 内部创建一个目录，用于存储桌面应用程序配置。本例中将其命名为 `ui`。

### 2. 添加 dsmuidir 键

在 `INFO` 或 `INFO.sh` 文件中添加 `dsmuidir` 键，其值为上一步创建的目录的相对路径。

```ini
dsmuidir="ui"
```

如果您有多个应用程序，应使用第二种格式：

```ini
dsmuidir="MyApp1:appui1 MyApp2:appui2"
```

在上述示例中，`MyApp1` 表示标识符，`appui1` 表示相对路径。

### 3. 软链接创建

套件安装后，DSM 会在 `/usr/syno/synoman/webman/3rdparty/[标识符]/` 创建相应的软链接，指向相对路径所在的位置。如果使用第一种格式（未指定标识符），DSM 将默认使用套件名称作为标识符。

### 4. 创建配置文件

根据需要，在 `dsmuidir` 指定的目录下创建应用程序配置和帮助配置。

### 5. 添加 dsmappname 键

在 `INFO` 或 `INFO.sh` 文件中添加 `dsmappname` 键，其值为应用程序配置中的唯一应用程序名称。当用户在套件中心点击套件的打开按钮时，将启动此应用程序。

```bash
dsmappname="com.company.App1"
```

---

## 配置说明

| 配置项 | 描述 |
|--------|------|
| `dsmuidir` | 指定桌面应用程序配置文件的存放目录 |
| `dsmappname` | 指定套件中心打开按钮对应的目标应用程序 |
| 应用程序图标 | 可在配置中自定义 |
| 应用程序权限 | 可设置不同用户角色的访问权限 |
| 目标 URL | 可配置应用程序打开的地址 |
