# 桌面应用

你可以为你的套件提供应用配置，这样配置好的应用就会显示在桌面菜单上。可以自定义图标、应用权限和目标 URL。

为了区分不同角色的用户，一个套件甚至可以提供多个应用，例如为管理员提供的管理应用和为普通用户提供的普通应用。

**DSM Web 图形界面**

此外，任何应用都可以通过提供帮助配置，将其自身的帮助文档带入桌面。

**DSM Web 帮助**

**设置桌面应用的步骤**

1.  在 `package.tgz` 内创建一个目录，该目录将用于存放桌面应用配置。例如，我们将其命名为 `ui`。

2.  在 `INFO` 或 `INFO.sh` 中添加 `dsmuidir` 键，其值为上一步创建的目录的相对路径。
    ```
    dsmuidir="ui"
    dsmuidir="MyApp1:appui1 MyApp2:appui2"
    ```
    如果你有多个应用，应使用第二种形式。在上面的示例中，`MyApp1` 代表一个标识符，`appui1` 代表一个相对路径。

3.  一旦套件安装完成，DSM 会在 `/usr/syno/synoman/webman/3rdparty/[identifier]/` 创建相应的软链接，链接到你的相对路径所在的位置。当使用第一种形式未指定标识符时，DSM 将默认使用套件名称作为标识符。

4.  如有需要，在 `dsmuidir` 指定的目录下创建你自己的应用配置和帮助配置。

5.  在 `INFO` 或 `INFO.sh` 中添加 `dsmappname` 键，其值为应用配置中唯一的应用名称。当在套件中心点击套件的“打开”按钮时，该应用将成为目标应用。
    ```
    dsmappname="com.company.App1"
    ```
