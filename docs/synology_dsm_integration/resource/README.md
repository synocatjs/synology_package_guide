# 资源

即使套件使用较低权限身份运行，也可以通过应用此机制来获取系统资源。

## 设置资源配置的步骤

1. **从资源列表中查找您需要的资源**
   - 查看可用的系统资源类型

2. **检查所选资源的相应时机是否满足**
   - 确保在正确的生命周期阶段请求资源

3. **在 `conf/resource` 创建文件并配置所需设置**

```json
{
    "data-share": {
        "shares": [
            {
                "name": "MyShareFolderName",
                "permission": {
                    "ro": ["MyUserName"]
                }
            }
        ]
    }
}
```

## 配置说明

| 字段 | 描述 |
|------|------|
| **data-share** | 资源类型（数据共享文件夹） |
| **shares** | 共享文件夹配置列表 |
| **shares[].name** | 共享文件夹名称 |
| **shares[].permission** | 权限设置 |
| **permission.ro** | 只读权限，指定可访问的用户列表 |
| **permission.rw** | 读写权限（如需读写访问） |

---

## 处理资源请求的实例

处理资源请求的实例称为 **worker**。当套件需要访问系统资源时，系统会创建一个 worker 来处理该请求，确保资源的安全访问。
