## 数据共享

### 描述

此 worker 在套件启动时创建共享文件夹并设置其权限。共享名称可以在规范中硬编码。共享文件夹在套件卸载后不会被删除，因为这可能会同时删除用户的个人数据。

---

### 行为说明

#### Acquire()（获取资源）
- 创建共享文件夹并设置其权限
- 如果共享文件夹已存在，跳过创建并直接设置权限

#### Release()（释放资源）
- 不执行任何操作

### 语法

```json
"data-share": {
  "shares": [{
    "name": "<share-name>",
    "permission": {
      "ro": ["<user-name>", ...],
      "rw": ["<user-name>", ...]
    },
    "once": "<once>"
  }, ...]
}
```

| 成员 | 自版本 | 描述 |
|------|--------|------|
| `shares` | 6.0-5914 | 对象数组，要创建的共享文件夹列表 |
| `name` | 6.0-5914 | 字符串，共享文件夹名称 |
| `permission` | 6.0-5914 | JSON 对象，共享文件夹的权限（可选） |
| `ro` | 6.0-5914 | 字符串数组，分配只读权限的用户 |
| `rw` | 6.0-5914 | 字符串数组，分配读写权限的用户 |
| `once` | 6.0-5914 | 布尔值，仅在套件首次启动时尝试创建共享文件夹（可选，默认 `false`） |

---

### 示例

以下规范创建名为 `music` 的共享文件夹，并为用户 `AudioStation` 分配只读权限。由于 `once` 默认为 `false`，上述过程在套件每次启动时都会执行。

```json
"data-share": {
  "shares": [{
    "name": "music",
    "permission": {
      "ro": ["AudioStation"]
    }
  }]
}
```

---

### 新增功能（自 7.0-41201）

从 DSM 7.0-41201 开始，套件中心会在 `/var/packages/[package_id]/shares/` 下创建以共享文件夹命名的符号链接，指向共享文件夹的实际路径。

---

### 使用场景

- 多媒体套件创建媒体库文件夹
- 备份套件创建备份目标文件夹
- 文件同步套件创建同步目录
- 为其他套件提供共享数据存储空间

---

### 注意事项

- 共享文件夹在套件卸载后**不会被删除**，以保护用户数据
- 如果共享文件夹已存在，只设置权限，不重新创建
- 使用 `once: true` 可避免在后续启动时重复检查共享文件夹
- 共享文件夹名称在系统中必须唯一
