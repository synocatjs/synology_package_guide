# INFO 文件字段说明

### 字段名称：package

**描述：** 套件标识。最终用户的 DSM 中同一时间只能存在一个版本的套件，因此该标识用于唯一识别您的套件。此外，套件中心会创建 `/var/packages/[套件标识]` 文件夹来存放套件文件。

**注意：** 此键值不能包含以下任何特殊字符：`:`、`/`、`>`、`<`、`|` 或 `=`。

**值类型：** 字符串

**默认值：**（空）

**示例：**
```
package="DownloadStation"
```

**DSM 版本要求：** 2.0-0731

---

### 字段名称：version

**描述：** 套件版本。最终用户可以识别套件版本。

**注意：**
- 每个版本分隔符仅允许使用 `.`、`-` 或 `_`
- 由分隔符分隔的每个版本号仅允许为数字
- 版本值格式应为 `[功能版本号]-[构建版本号]`。每次版本更新都应增加构建版本号，该号码可用于区分不同 DSM 主版本之间的套件版本

**值类型：** 字符串，主版本号、次版本号、修订版本号和构建版本号应在 0 到 2³¹-1 范围内

**默认值：**（空）

**示例：**
```
version="3.6-3263"
version="1.2.3-0001"
```

**DSM 版本要求：** 2.0-0731

---

### 字段名称：os_min_ver

**描述：** 运行套件所需的最低 DSM 版本。在 DSM 7.0 之后，该值应至少为 7.0-40000。

**值类型：** X.Y-Z（DSM 主版本号、DSM 次版本号、DSM 构建版本号）

**默认值：** 无

**示例：**
```
os_min_ver="7.0-40000"
```

**DSM 版本要求：** 6.1-14715

---

### 字段名称：description

**描述：** 套件中心显示的套件简短描述。

**值类型：** 字符串

**默认值：**（空）

**示例：**
```
description="Download Station 是一个基于网页的下载应用程序，允许您通过 BT、FTP、HTTP、NZB、迅雷、FlashGet、QQDL 和 eMule 从互联网下载文件，并订阅 RSS 源以获取最新热门或最新 BT 资讯。它提供自动解压服务，帮助您在文件下载后解压缩到 Synology NAS。借助 Download Station，您可以从多个文件托管站点下载文件，并通过系统默认搜索引擎以及通过 BT 搜索功能自添加的引擎搜索种子文件。"
```

**DSM 版本要求：** 2.3-1118 / 4.2-3160

---

### 字段名称：arch

**描述：** 列出可以安装套件的 CPU 架构列表。

**值类型：** 架构值之间用空格分隔。更多信息请参阅附录 A：平台和架构值映射表

**默认值：** `noarch`

**注意：**
- `noarch` 表示套件可以在任何平台上安装和运行。例如，用 PHP 或 Shell 脚本编写的套件
- 请不要将不同平台的二进制文件打包到同一个 SPK 文件中

**示例：**
```
arch="noarch"
```
或
```
arch="x86_64 alpine"
```

**DSM 版本要求：** 2.0-0731

---

### 字段名称：maintainer

**描述：** 套件中心显示的套件开发者信息。

**值类型：** 字符串

**默认值：**（空）

**示例：**
```
maintainer="Synology Inc."
```

**DSM 版本要求：** 2.0-0731
