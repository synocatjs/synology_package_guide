# INFO 文件：必填字段

| 字段名 | 描述 | 值 / 格式 | 默认值 | 示例 | DSM 版本要求 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **`package`** | **套件标识符。** 用于在 DSM 中唯一标识您的套件。系统会以此名称在 `/var/packages/` 下创建文件夹。 | 字符串。<br>**注意：** 不能包含 `:` `/` `>` `<` <code>\|</code> `=` 这些特殊字符。 | (空) | `package="DownloadStation"` | 2.0-0731 |
| **`version`** | **套件版本号。** 格式为 `[功能版本号]-[构建版本号]`。每次发布都应增加构建版本号。 | 字符串。版本号只能使用 `.` `-` `_` 作为分隔符，且版本号部分只能为数字。<br>范围：`0` 到 `2^31-1`。 | (空) | `version="3.6-3263"`<br>`version="1.2.3-0001"` | 2.0-0731 |
| **`os_min_ver`** | **最低 DSM 版本要求。** 套件正常运行所需的最低 DSM 版本。DSM 7.0 后此值应至少为 `7.0-40000`。 | `X.Y-Z` 格式<br>（DSM 主版本号.次版本号-构建号） | 无 | `os_min_ver="7.0-40000"` | 6.1-14715 |
| **`description`** | **套件描述。** 在套件中心显示的简短描述。 | 字符串 | (空) | `description="Download Station 是一个基于网页的下载应用..."` | 2.3-1118<br>4.2-3160 |
| **`arch`** | **支持的 CPU 架构。** 列出套件可以安装的 CPU 架构类型。 | 字符串，多个值用空格分隔。<br>**注意：** `"noarch"` 表示兼容所有平台。 | `noarch` | `arch="noarch"`<br>`arch="x86_64 alpine"` | 2.0-0731 |
| **`maintainer`** | **维护者/开发者。** 在套件中心显示的开发者名称。 | 字符串 | (空) | `maintainer="Synology Inc."` | 2.0-0731 |
