# script 目录

脚本目录，包含包生命周期各阶段执行的脚本。

这是 Synology DSM 包的**脚本目录**，包含包生命周期各阶段执行的脚本。让我详细解释每个脚本的作用：

## 📋 脚本文件详解

### 🔄 安装/升级阶段

| 脚本 | 触发时机 | 主要用途 |
|------|----------|----------|
| **preinst** | 安装前 | 检查依赖、准备环境 |
| **postinst** | 安装后 | 配置服务、初始化数据 |
| **preupgrade** | 升级前 | 备份配置、停止服务 |
| **postupgrade** | 升级后 | 恢复配置、迁移数据 |

### 🗑️ 卸载阶段

| 脚本 | 触发时机 | 主要用途 |
|------|----------|----------|
| **preuninst** | 卸载前 | 停止服务、清理资源 |
| **postuninst** | 卸载后 | 删除数据、清理链接 |

### ⚙️ 运行时

| 脚本 | 触发时机 | 主要用途 |
|------|----------|----------|
| **start-stop-status** | 启动/停止 | 控制服务生命周期 |

在示例中，除 start-stop-status 和 postinst 外，其他脚本均为 `exit 0`。这些脚本可以根据实际需求进行编写，以实现特定的功能。
